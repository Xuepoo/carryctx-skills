# Test-Driven Development (TDD) SOP

## 1. RED Phase

- Understand the requirement.
- Write the minimal test case that will fail based on the new requirement.
- Run the test and explicitly confirm it fails.

## 2. GREEN Phase

- Write the simplest code necessary to pass the test. Do not over-engineer.
- Run the test suite and confirm everything passes.

## 3. REFACTOR Phase

- Clean up the code. Remove duplication.
- Ensure tests still pass after every refactoring step.

## 4. Repeat

- Move to the next requirement or edge case and repeat from step 1.
